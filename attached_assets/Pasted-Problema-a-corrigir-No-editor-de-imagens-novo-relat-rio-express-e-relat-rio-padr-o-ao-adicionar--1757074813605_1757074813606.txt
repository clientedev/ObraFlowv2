Problema a corrigir:
No editor de imagens (novo relatório express e relatório padrão), ao adicionar a ferramenta Texto com Fabric.js, o objeto aparece na tela, mas o teclado não abre no mobile para que o usuário possa digitar e editar o conteúdo.

O que fazer:

Analisar o código da tela de edição de imagem onde o objeto fabric.IText ou fabric.Textbox é criado.

Corrigir para que, ao adicionar um objeto de texto:

O objeto seja criado como fabric.IText (e não apenas fabric.Text), pois somente IText tem suporte nativo a edição interativa com teclado.

Ao selecionar o objeto de texto no mobile, forçar o foco em um input hidden ou no próprio objeto IText, garantindo que o teclado virtual do dispositivo apareça.

Usar canvas.setActiveObject(text) + text.enterEditing() + text.hiddenTextarea.focus() para ativar o modo de edição.

Garantir que text.exitEditing() funcione ao clicar fora ou confirmar.

Caso necessário, adicionar um textarea invisível (com opacity:0; position:absolute; z-index:-1;) para capturar o foco no mobile e sincronizar o valor com o objeto IText.

Exemplo de ajuste sugerido (Fabric.js):

const text = new fabric.IText('Digite aqui', {
  left: 100,
  top: 100,
  fontSize: 20,
  fill: '#000',
  editable: true,
});
canvas.add(text);
canvas.setActiveObject(text);
text.enterEditing();

// Forçar foco no mobile
if (text.hiddenTextarea) {
  text.hiddenTextarea.focus();
}


Verificar se o CSS não está bloqueando o textarea interno do Fabric.js. Ele precisa estar presente no DOM para que o teclado apareça.

Se o teclado ainda não abrir no mobile, criar um input[type="text"] temporário invisível que recebe foco (focus()) e sincroniza valor com o objeto IText.

Critério de aceitação:

Ao adicionar a ferramenta Texto no editor de imagens, o usuário toca no texto → o teclado aparece imediatamente no celular → é possível escrever normalmente.

Funciona tanto no novo relatório express quanto no relatório padrão.

Nenhuma outra funcionalidade do sistema é alterada (todo o restante já está 100% funcional).
